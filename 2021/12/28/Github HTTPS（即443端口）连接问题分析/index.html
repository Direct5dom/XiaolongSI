Github HTTPS（即443端口）连接问题分析

Posted on 2021-12-28 Edited on 2021-12-31 In 网络

连接状态简述

根据测试log（见附件）可知，当用户未尝试访问页面时，可以连接到端口，Ping返回延迟。当用户尝试打开页面的一瞬间，端口被关闭，同时Ping返回超时（在附件中则为2000ms+的延迟），此时页面表现为打不开、连接超时。

而端口被关闭的时间约为3分钟，3分钟过后端口再次可建立TCP连接，直到下次尝试访问页面。

关于“断断续续”可以打开

参考测试log（见附件）可知，当触发“端口关闭”这一行为后并不能保证连接被完全阻断。在一系列连接超时中会有几次成功连接，这足以让网站可以“断断续续”的成功打开。同时打开网站也会调用本地缓存，缓存的内容也会辅助页面的渲染营造出成功打开的伪象。
